# 프롬프트 피드백 히스토리

typo-vibe 작업하면서 매일 요청→결과물 회고에서 나온 내용 누적.
이 파일은 다음 요청 때 AI가 더 빨리 방향을 잡도록 돕는 기준점.

---

## 1. 요청 시 필수 3요소

레퍼런스/짤 재현 같은 시각적 작업은 첫 메시지에 이 3개를 박아두면
왕복 줄어듦.

1. **레퍼런스 이미지 첨부** — 짤/디자인 캡처 그대로. 말로 묘사하지 말 것.
2. **재현 vs. 변주 비율** — "원본 90% 재현 + 인터랙션 10%" 처럼 숫자로.
3. **기술 예산** — 브라우저만 / API 써도 됨 / 서버 필요 가능 중 명시.

이게 없으면 AI가 "업로드가 본질인가, 재현이 본질인가" 헷갈려서 엉뚱한
방향으로 먼저 구현함. day06에서 실제로 그렇게 돌아감.

---

## 2. 작업 지시 문법

### 구체적으로 쓸 것 (day05 BEEF에서 검증된 패턴)

```
day0X 폴더에 [주제] 인터랙티브 페이지 만들어줘.

컨셉: [한 문장 요약 — 톤+비주얼이 한 번에 잡히게]

인터랙션:
- 마우스로 [동사]
- 휠로 [동사]
- [특정 조건]에서 [결과]

숨은 요소:
- [특정 좌표]에 [효과]
- 메시지는 이런 문장들: "..."

부가: [음악/HUD/기타]
끝나면 메인 아카이브에 카드 추가.
```

핵심:
- **위치 명시**: `day0X/`, `파일명.jpg 같은 폴더`
- **인터랙션은 동사 단위 불릿**
- **"숨은 요소"는 따로 분리** — 진짜 핵심 서사가 거기 있다는 신호
- **구체 문구 예시 1~2개** — 나머지 톤이 따라감
- **수치 범위 명시** — "적당히" 대신 "0.7~3배", "3곳"
- **후속 작업 포함** — 아카이브 등록까지

### 미니멀 버전도 가능
위치 + 주제 + 인터랙션 1~2개 + 대표 문장 하나면
80~90% 결과물이 한 번에 나옴.

---

## 3. "히스토리 살려줘" 같은 모호 표현

이전 세션 기억 못 함. workspace에 흔적 있으면 복원 시도하지만
흔적 없으면 재질문 받음.

명확한 대안:
- "day0X에 만들어둔 거 아카이브에 카드로 추가해줘"
- "지난번 X 작업 이어서 Y 추가해줘"

"일전에 얘기했던" 류 레퍼런스는 가능하면 피하고,
**workspace 내 찾을 단서**(파일명, 폴더명)로 대체.

---

## 4. 커뮤니케이션 선호

- **솔직 우선** — 포장/과장/사탕발림 안 좋아함. 결과물이 별로면 "별로"라고
  바로 말하는 걸 선호. AI도 같은 톤으로 답할 것.
- **장황한 설명 < 결과물 보여주기** — 설명보다 일단 만들어서 결과 공유.
- **중간 점검 좋아함** — 크게 갈리는 결정 지점에선 합의하고 가기.
  ("A/B/C 중 어느 쪽?" 형태 질문 OK)
- **프롬프트 피드백 자주 요청** — "이 요청 어떻게 더 잘 쓸 수 있었어?"
  같은 회고를 자주 함. 답할 때 **구체 예시**로 답할 것.

## 5. 실패 유형 — 이런 건 미리 말해줄 것

### 기술적으로 막히는 지점
- **정적 사이트에서 이미지 생성 AI** — Stable Diffusion img2img 등.
  API 키 브라우저 노출 문제. 서버 필요.
- **업로드 사진을 임의 구도/포즈로 변환** — 단순 AI 세그멘테이션으로는
  배경 제거까지만 가능. 자세·방향 바꾸기는 생성 모델 필요.
- **색상 매칭으로 "의미적 대응"** — RGB 거리만으론 "이 고양이 품종 →
  이 맹수" 매칭 정확도 한계 명확.

이런 게 요청에 섞여 있으면 **초반에 "이 부분은 현재 스택으로 어려움,
대안 A/B 중 선택"** 이라고 제시해야 함. 붙어있는 요구 중 하나만
AI 없이 가능하면 그 하나만 짚어서 "이건 되는데 저건 안 됨" 분리.

### 방향 혼동
- 재현 vs. 실험 비율 불명확 → 중간에 방향 틀어짐
- "사이드라 부담 없이" 해석 차이 → AI는 "기능 가볍게" 로 읽고
  사용자는 "완성도 감각은 유지" 로 쓴 경우 있음. 명시할 것.

## 7. 이번 day06 회고 (반면교사)

- 원본 짤 재현이 목표였는데 "업로드 기능 추가"가 메인 방향으로 튐
- vol.1(프리셋) → vol.2(업로드+색매칭) → vol.2(MediaPipe) → vol.2(실사 프리셋)
  로 3번 방향 틀음. 실패 반복.
- 이 과정에서 나온 실질 교훈:
  - "부담 없이"를 AI가 "기능 축소"로 해석 금지. 완성도는 별개.
  - 첫 메시지에 레퍼런스 이미지 있었으면 시작부터 방향 잡혔을 것.
  - "업로드 재미"와 "원본 재현" 둘 다 원할 때 기술적 공존 가능 여부를
    먼저 검증하고 가야 함.
